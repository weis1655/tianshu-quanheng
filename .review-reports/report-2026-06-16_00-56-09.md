## 1. 📋 业务逻辑问题 (Business Logic)
### Issue #1: MemPalace 子进程集成对错误/边界数据容忍性不足 [Confidence: Medium]
- **文件路径**：`tianshu_memory.py` [Line 42-62]
- **问题描述**：`TianshuMemory._kg_run` 通过拼接 Python 代码字符串并传给子进程（`subprocess.run([_MEMPALACE_PY, "-c", full], ...)`）来执行 KG 操作。随后在 `save_run_summary` 中，代码直接用来自 `pools` 的 `code`、`pool_name`、`entry_date` 等字段构建要执行的 Python 代码片段（通过 f-string 插入到 `code_block` 中）。当这些字段包含双引号、换行或恶意构造的字符串时，会导致子进程执行异常或意外注入执行路径，存在命令/代码注入风险并且对错误输入容忍性低。
- **影响分析**：可能导致远端命令注入或 MemPalace 子进程异常崩溃，破坏数据一致性；若外部数据未被严格信任，存在被利用的高风险（CWE-78 / CWE-94）。
- **建议方案**：不要直接拼接代码字符串注入子进程；改为传输结构化数据（JSON）并在子进程的 Python 脚本中通过安全接口（`json.loads`）调用 KG API，或使用直接的 Python API 导入 `mempalace`（避免 spawn 子进程）。
  ```diff
  - full = "\n".join([... f'kg.add_entity("{code}", "stock")', ...])
  - r = subprocess.run([_MEMPALACE_PY, "-c", full], capture_output=True, text=True, timeout=30)
  + payload = {"ops": [{"op": "add_entity", "args": [code, "stock"]}, ...]}
  + proc = subprocess.run([_MEMPALACE_PY, "-c", "import sys, json; sys.path.insert(0, '...'); from mempalace.knowledge_graph import KnowledgeGraph; kg = KnowledgeGraph(...); ops=json.load(sys.stdin);\nfor o in ops['ops']: getattr(kg, o['op'])(*o['args'])"], input=json.dumps(payload), text=True, capture_output=True, timeout=30)
  ```

#### Issue #2: [安全] 不安全的 `subprocess.run(..., shell=True)` 使用 - 中等
- **文件路径**：`agents/trigger.py` [Line ~30-80]
- **问题描述**：使用 `subprocess.run(cmd, shell=True, ...)` 执行 `curl` 字符串。当前 `cmd` 来自固定字符串，但未来若拼接用户输入（或配置文件）可能导致 shell 注入。建议使用 `requests`/`httpx` 直接替代 `curl`，或在确需子进程时使用列表参数并关闭 `shell=True`。
- **修复建议**：
  ```diff
  - cmd = 'curl -sL --max-time 10 "https://qt.gtimg.cn/q=flashing_china"'
  - r = subprocess.run(cmd, shell=True, capture_output=True, timeout=15)
  + import requests
  + resp = requests.get("https://qt.gtimg.cn/q=flashing_china", timeout=10)
  + content = resp.content.decode('gbk', errors='replace')
  ```
