## 一、修复报告规范

### 单工单交付标准

每个工单完成后，必须输出以下内容：

```
## 修复报告：{工单编号} — {标题}

### 改动说明
- 文件：{路径}（+N/-M 行）
- 逻辑：{一句话描述改了啥}
- 风险：{低/中/高} — {说明}

### 验证步骤
1. {步骤1}
2. {步骤2}
3. {步骤3}

### 测试结果
- 编译：✅ python -m py_compile {文件}
- 导入：✅ python -c "import {模块}"
- 逻辑：✅ {验证通过的场景}
- 回归：✅ {受影响的其他模块验证}

### 改动文件清单
- [ ] {文件路径} — 已修改
- [ ] {文件路径} — 已修改
```

### 断点续修规则

```
1. 每批次完成 1 个工单后，保存 checkpoint 到 data/项目管理/checkpoint_{批次}.json
2. 重启时先读取 checkpoint，校验已完工单：
   - git log --oneline 检查是否合入 main
   - python -m py_compile 检查文件是否可编译
   - 如未合入/编译失败 → 标记为「待重做」
3. 未完成工单按原顺序自动接续，上下文从工单描述重新加载
4. 已完工单但未通过验证的，标记为「待修复-回归」加入下一批次
```

### WO-002 评分衰减加速 — 0分标的3天强制降级

**类型**：算法逻辑修复
**预估工时**：0.5人·天
**依赖**：WO-001（需用 DOWNGRADE_THRESHOLD）

**问题**：当前评分衰减每天0.5分，15分封顶。评分0分的标的要30天才会自然低于65分被降级。回头看报告显示海光信息(0分)、阳光电源(0分)、光线传媒(0分)等评分归零的标的长期滞留。

**改动**：在 pool_manager.py 的 _scan_and_downgrade() 中增加规则：评分=0且入池≥3天的标的，直接降级，不等待衰减。

```python
# 新增规则（在现有 score < THRESHOLD 判断之前）
if score <= 0:
    try:
        entry_date = s.get("纳入日期", "")
        if entry_date:
            days = (datetime.now() - datetime.strptime(entry_date, "%Y-%m-%d")).days
            if days >= 3:
                to_demote.append(s)
                continue
    except ValueError:
        pass
```

**验收标准**：
```
□ 模拟：评分0分、入池3天的标的 → 被降级
□ 模拟：评分0分、入池1天的标的 → 保留（给评分恢复机会）
□ 现有评分≥65的标的 → 不受影响
□ sweep_downgrade.py 验证通过
```

### WO-003 bare except 治理 — 分级处理

**类型**：代码健壮性
**预估工时**：1人·天
**依赖**：无

**问题**：15处 `except: pass` 分布在 main.py、review_agent.py、pool_updater.py、feedback_loop.py 等。pool_updater.py L106-107 的 `except: pass` 尤其危险——静默吞掉所有异常，磁盘满/API崩了都不知道。

**分级处理规则**：

| 异常类型 | 处理方式 | 示例 |
|----------|----------|------|
| 火警（LLM失败/API限流/数据持久化失败） | 必须 `print(f"❌ {e}")` + `record_failure()` | `except Exception as e: print(f"❌ 持久化失败: {e}")` |
| 黄牌（单只股票行情失败/非关键路径） | 必须 `print(f"⚠️ {e}")` | `except Exception as e: print(f"⚠️ 行情刷新失败: {e}")` |
| 绿牌（文件不存在/缓存过期） | 可静默但需注释 | `except FileNotFoundError: pass  # 缓存未生成，合逻辑` |

**改动范围**：逐文件扫描，14处 bare except 逐一升级。

**验收标准**：
```
□ grep -rn "except:" agents/*.py | grep -v "except Exception\|except FileNotFoundError\|except ValueError\|except TypeError\|except KeyError\|except ImportError\|except OSError\|except json.JSONDecodeError\|except StopIteration"
  返回 0 条
□ 所有火警级异常有 print(f"❌ ...") + record_failure()
□ 模拟API限流 → 日志有清晰错误记录，不静默
```
