# 第一阶段：风控体系全景审计报告

**审计日期：** 2026-07-14
**审计范围：** 事前风控/事中风控/合规风控/异常风控 全链路
**方法：** 三维度并行审计 + 直接验证

---

## 一、风控全链路架构

### 事前风控（入场前）

1. **标的筛选风控** → review_agent.py (ST/市值/评分过滤), screen_agent.py (快筛池)
2. **涨跌停拦截** → decision_agent._filter_limit_up() (涨停/跌停排除)
3. **硬规则校验** → decision_agent R1-ST禁入 / R2-流通市值<5亿 / R3-换手率>30% / R4-评分<55
4. **二审制Gate** → gate_controller.py (Skeptic质疑阻塞→连续3次自动降级)
5. **重复推荐保护** → _filter_duplicate_recommendations (7天过滤)
6. **仓位建议** → LLM prompt中有但代码无强制

### 事中风控（运行中）

1. **熔断器系统** → error_handling.py (5个熔断器: news/screen/review/decision/skeptic)
2. **极端行情降级** → _get_market_state 创业板-3%→规则决策
3. **弱市模式** → market_decision_context 动态仓位/跳过LLM
4. **连续失败监控** → 熔断器半开/全开状态

### 事后风控（闭环）

1. **T+1追踪** → closed_loop_tracker.py (次日3日涨跌)
2. **回头看报告** → scripts/回头看.py (定期复盘)
3. **反馈闭环** → feedback_loop.py

### 缺失风控（审计发现）

1. **❌ 仓位强制执行** — 只在prompt中写"单票≤30%"，无代码级校验
2. **❌ 止损止盈自动执行** — 只在prompt中建议，无自动触发
3. **❌ 总仓位上限** — 无全局仓位管理
4. **❌ 单日开仓限额** — 无限制
5. **❌ 实盘浮盈浮亏监控** — 仅在回顾中事后计算
6. **❌ 池容量溢出保护** — 边缘池满时截断逻辑存在但早期版本未触发
