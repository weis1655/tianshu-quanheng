# 多策略组合管理模块 — 任务拆解与测试用例设计

## 一、开发任务拆解表

### 批次1（P0核心功能）

#### PM-001: 策略池管理完善

| 项目 | 内容 |
|:----|:------|
| **关联功能** | F1.4 版本管理 + F1.5 状态监控 |
| **实现方案** | 扩展 StrategyConfig 数据模型，新增 versions[]、metrics字典、status字段；新增 `get_version_history()`、`rollback_version()`、`_auto_version()` 方法；新增 `get_strategy_status()` 状态推导逻辑 |
| **核心逻辑** | 每次 `update_strategy()` 自动递增版本号；status 根据 metrics.drawdown/max_drawdown 推导 active/warning/circuit_triggered/stopped |
| **联调依赖** | 无（独立模块） |
| **预计工时** | 1.0d |
| **关键代码变化** | StrategyConfig 新增字段；PortfolioManager 新增 4 个方法；持久化扩展 |

#### PM-002: 多层级资金分配

| 项目 | 内容 |
|:----|:------|
| **关联功能** | F2.5 策略→标的层级分配 |
| **实现方案** | 新增 `allocate_multi_tier()` 方法：先按 allocation 方法分配策略层资金，再按评分权重分配单票仓位 |
| **核心逻辑** | `strategy_capital = total_capital × w_i`；`position_capital = strategy_capital × (score_j / Σscore_in_strategy)` × max_position_pct |
| **联调依赖** | PoolManager（读持仓池数据） |
| **预计工时** | 0.5d |
| **关键代码变化** | 新增 allocate_multi_tier()；对接 load_pool() |

#### PM-003: 相关性监控

| 项目 | 内容 |
|:----|:------|
| **关联功能** | F3.1 策略间相关性监控 |
| **实现方案** | 新增 `update_correlation()` + `check_correlation_alerts()`；EWMA波动率估计；滚动30天收益率窗口；Pearson相关系数计算 |
| **核心逻辑** | `r_xy = Σ(x_i-μ_x)(y_i-μ_y) / sqrt(Σ(x_i-μ_x)² × Σ(y_i-μ_y)²)`；r>0.7告警，r>0.85强制再平衡 |
| **联调依赖** | 绩效历史数据（需要策略日收益率） |
| **预计工时** | 1.0d |
| **关键代码变化** | 新增2个方法；新增 correlation_matrix 缓存；correlation_cache.json 持久化 |

#### PM-004: 策略熔断机制

| 项目 | 内容 |
|:----|:------|
| **关联功能** | F3.2 单策略最大回撤熔断 |
| **实现方案** | 新增 `check_strategy_circuit_breaker()`；当 metrics.drawdown ≤ max_drawdown → 熔断暂停；恢复：drawdown 回升至 max_drawdown×0.5 → 自动恢复 |
| **核心逻辑** | `drawdown_ratio = current_drawdown / max_drawdown`；≤1→熔断，>0.5→恢复；状态持久化 |
| **联调依赖** | 无（读自身 metrics） |
| **预计工时** | 0.5d |
| **关键代码变化** | 新增2个方法；circuit_state.json 持久化 |

#### PM-005: 总仓位/容量/跨策略约束

| 项目 | 内容 |
|:----|:------|
| **关联功能** | F3.3+F3.4+F3.5 总仓位约束、容量上限、跨策略暴露 |
| **实现方案** | 增强 `check_portfolio_risk()`：新增总持仓数检查、跨策略同标的聚合检查；新增 `check_capacity_limits()` |
| **核心逻辑** | 跨策略暴露：遍历所有策略持仓，按代码聚合，单标的跨策略仓位 > 30% 告警；总持仓 > 30 告警 |
| **联调依赖** | PoolManager（读全池数据） |
| **预计工时** | 0.5d |
| **关键代码变化** | check_portfolio_risk 增强；新增 cross_strategy_exposure 检查 |

#### PM-006: 定期+平滑调仓

| 项目 | 内容 |
|:----|:------|
| **关联功能** | F4.1+F4.3 定期再平衡、平滑调仓 |
| **实现方案** | 新增 `periodic_rebalance()` 按配置周期执行；新增 `_smooth_adjustment()` 限制单次调仓幅度；新增 rebalance_cooldown 防抖 |
| **核心逻辑** | 单次最大调仓 ≤ max_turnover_per_trade(20%)；超额部分分3天执行；防抖1小时 |
| **联调依赖** | 无 |
| **预计工时** | 0.5d |
| **关键代码变化** | 新增3个方法；config.yaml 读取周期配置 |

#### PM-007: 优胜劣汰机制

| 项目 | 内容 |
|:----|:------|
| **关联功能** | F4.4 策略优胜劣汰 |
| **实现方案** | 新增 `survival_competition()`：每 N 天评估，综合评分=Sharpe×0.5 + WinRate×0.3 + maxDrawdown×0.2(反向)；淘汰末位，释放资金 |
| **核心逻辑** | `score = sharpe_norm×0.5 + win_rate_norm×0.3 + (1-drawdown_norm)×0.2`；可淘汰数=1；释放资金等分给其余策略 |
| **联调依赖** | 绩效历史数据 |
| **预计工时** | 1.0d |
| **关键代码变化** | 新增 survival_competition()；淘汰后自动 rebalance |

#### PM-010: 配置扩展

| 项目 | 内容 |
|:----|:------|
| **关联功能** | 所有功能（配置参数化） |
| **实现方案** | 扩展 config.yaml 新增 `portfolio:` 节；PortfolioManager 初始化时从 config_loader 读取；提供 `_load_portfolio_config()` 方法 |
| **核心逻辑** | 配置优先级：代码默认值 < config.yaml < 运行时参数；自动回退 |
| **联调依赖** | config_loader.py |
| **预计工时** | 0.3d |
| **关键代码变化** | config.yaml 扩展；PortfolioManager.__init__ 加载配置 |
