## 常见故障模式总结

### 模式1：格式漂移（最频繁，占P0的30%+）

**症状**：LLM输出格式变化→正则提取失败→score=0→级联异常

**典型链路**：
```
LLM prompt/模型更新 → 输出格式变化 → review/screen正则失败 → score=0
→ 要么降级延迟（score=0但flow非降级）
→ 要么格式漂移假阳性（回头看报告误报23个P0）
```

**防护**：
1. 正则加fallback链（至少3级）
2. 对score=0强制降级
3. 回头看报告加假阳性检测

### 模式2：降级延迟（占P0的25%+）

**症状**：低分标的滞留在池中未被清理

**根因**：
1. 评分调整后降级检查未触发（流程控制缺失）
2. 评分衰减累加bug（衰减公式错误）
3. 全池低分扫描未覆盖所有池

**防护**：
1. 评分调整后必须跟降级检查
2. sweep_downgrade.py独立扫描
3. 所有池都纳入低分扫描范围

### 模式3：LLM退化（占P0的15%+）

**症状**：LLM输出无交易方案的退化报告

**根因**：max_tokens不足/模型版本变化/prompt退化

**防护**：
1. max_tokens=3000+
2. 输出有效性校验（【主推】+止损+止盈）
3. 连续退化计数器→force_rule_mode
