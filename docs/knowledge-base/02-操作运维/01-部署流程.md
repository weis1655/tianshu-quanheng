## 一、概述

本文档描述从零开始部署天枢权衡（Tianshu Quanheng）系统的完整流程，包括环境准备、依赖安装、配置修改、启动验证和常见问题处理。适用于首次部署或迁移到新环境时参考。

**系统概要：** 天枢权衡是一个基于 LLM 的多智能体 A 股选股与决策分析系统。串行流水线运行：新闻 → 快筛 → 审查 → SkepticGate → 决策 → 复盘，配合五池流转（快筛候选池/重点观察池/S级操作池/边缘池/持仓池）。

## 二、前置条件

### 2.1 硬件要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| CPU | 2 核 | 4 核及以上 |
| 内存 | 4 GB | 8 GB+ |
| 磁盘 | 5 GB 可用空间 | 20 GB+（含历史数据） |
| 网络 | 公网访问 | 稳定宽带，无需代理直连 |

### 2.2 软件要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| **操作系统** | Linux (Ubuntu 22.04+/Debian 12+) / macOS | Windows 未测试 |
| **Python** | 3.11+ | 实测环境 Python 3.11.15 |
| **pip** | 23.0+ | Python 包管理器 |
| **Git** | 2.30+ | 克隆仓库 |
| **Hermes Agent** | 最新版 | 用于 Cron 调度和技能管理 |

### 2.3 网络要求

- 可访问 GitHub（`https://github.com/weis1655/tianshu-quanheng`）克隆仓库
- 可访问 LLM API 端点（OpenCode Zen / SenseNova / 其他 OpenAI 兼容 API）
- 可访问股票行情数据源（Sina Finance / 东方财富等）
- 可访问新闻源（DuckBurn GitHub / CCTV / 东方财富等）

> ⚠️ 系统 HTTP 代理（如 `192.168.197.109:7897`）会导致 GitHub 直连 SSL 超时。部署时建议设置 `no_proxy=github.com` 或直接绕过代理。

## 三、部署步骤

### 3.1 克隆仓库

```bash
# 进入项目存放目录
cd ~/hermes-data/

# 克隆仓库
git clone https://github.com/weis1655/tianshu-quanheng.git
cd tianshu-quanheng
```

> ⚠️ 仓库为私有仓库，需要配置 GitHub 认证后访问。如果使用 `gh` CLI：
> ```bash
> gh auth login
> gh repo clone weis1655/tianshu-quanheng
> ```

### 3.2 创建 Python 虚拟环境

```bash
# 确保 Python 3.11+
python3 --version  # 应显示 Python 3.11.x

# 创建虚拟环境（推荐在项目目录外，避免与 git 冲突）
python3 -m venv ~/hermes-data/tianshu-quanheng-venv

# 激活虚拟环境
source ~/hermes-data/tianshu-quanheng-venv/bin/activate

# 验证 python 路径
which python  # 应指向虚拟环境路径
```

> 💡 也可以使用项目内 `.venv` 目录（已加入 `.gitignore` 无需额外配置）：
> ```bash
> python3 -m venv .venv
> source .venv/bin/activate
> ```

### 3.3 安装依赖

```bash
# 确保在虚拟环境激活状态下
pip install --upgrade pip
pip install -r requirements.txt
```

**核心依赖清单：**

| 包名 | 版本要求 | 用途 |
|------|---------|------|
| `requests` | >=2.28.0 | HTTP 请求 |
| `pyyaml` | >=6.0 | YAML 配置解析 |
| `pandas` | >=2.0.0 | 数据处理 |
| `numpy` | >=1.24.0 | 数值计算 |
| `httpx` | >=0.24.0 | 异步 HTTP 客户端 |
| `aiohttp` | >=3.8.0 | 异步 HTTP 支持 |
| `python-json-logger` | >=2.0.0 | 结构化日志 |

**可选依赖：**

| 包名 | 用途 | 安装命令 |
|------|------|---------|
| `apscheduler` | 完整调度功能 | `pip install apscheduler>=3.10.0` |
| `pytest` | 单元测试 | `pip install pytest>=7.0.0 pytest-cov pytest-asyncio` |
| `black` | 代码格式化 | `pip install black>=23.0.0` |
| `flake8` | 代码检查 | `pip install flake8>=6.0.0` |
| `mypy` | 类型检查 | `pip install mypy>=1.0.0` |

> ⚠️ 依赖不包含 `dotenv`，但系统支持 `python-dotenv` 以自动加载 `.env` 文件。建议安装：
> ```bash
> pip install python-dotenv
> ```

### 3.4 配置环境变量

创建或编辑 `~/.hermes/.env` 文件，添加 LLM API 凭据：

```bash
cat >> ~/.hermes/.env << 'EOF'
# LLM API 配置
OPENCODE_ZEN_API_KEY=sk-your-key-here
SENSENOVA_API_KEY=sk-your-sensenova-key-here

# 可选：邮件 SMTP 配置
SMTP_USER=your_email@example.com
SMTP_PASSWORD=your_smtp_password
SMTP_SERVER=smtp.example.com
SMTP_PORT=465
EOF
```

> ⚠️ 环境变量加载顺序：`main.py` 启动时优先从 `~/.hermes/.env` 加载，其次是项目根目录 `.env` 文件。`~/.hermes/.env` 是 Hermes Agent 的标准配置位置。

验证环境变量是否生效：

```bash
# 在虚拟环境激活状态下
python3 -c "import os; print('OPENCODE_ZEN_API_KEY' in os.environ or '已配置')"
```

### 3.5 配置系统参数

系统配置文件为 `config.yaml`，位于项目根目录：

```bash
# 配置文件已存在，直接编辑
vim config.yaml
```

**关键配置项说明：**

| 配置项 | 默认值 | 说明 |
|--------|-------|------|
| `api.opencode_url` | `https://opencode.ai/zen/v1/chat/completions` | LLM API 端点 |
| `api.opencode_key` | `${OPENCODE_API_KEY:-***}` | 从环境变量读取 |
| `api.default_model` | `minimax-m2.5-free` | 默认 LLM 模型 |
| `api.llm.temperature` | `0.3` | LLM 温度参数 |
| `api.llm.max_tokens` | `1000` | 最大 Token 数 |
| `api.llm.timeout` | `60` | API 超时时间（秒） |
| `api.llm.max_retries` | `3` | 最大重试次数 |
| `api.llm.backend` | `sensenova` | 后端选择: `opencode`/`gemini`/`doubao`/`sensenova` |
| `pools.capacity.*` | 见下 | 各池容量限制 |
| `schedule.timezone` | `Asia/Shanghai` | 时区 |
| `features.auto_schedule` | `true` | 启用自动调度 |
| `features.feishu_notify` | `false` | 飞书通知开关 |

**池容量默认值：**

| 池名 | 默认容量 | 超限处理 |
|------|---------|---------|
| 快筛候选池 | 20 | 保留前 20 个，截断告警 |
| 重点观察池 | 20 | 保留最新 20 个，截断告警 |
| 边缘池 | 30 | 保留最新 20 个（实际受阈值限制） |
| S级操作池 | 3 | T+1 过期自动清理 |
| 持仓池 | 不限 | — |

> 💡 API 密钥不要直接写在 config.yaml 中，应通过环境变量注入。`${OPENCODE_API_KEY:-***}` 表示优先取环境变量，未设置时使用 `***`。

### 3.6 创建运行时目录

系统首次运行时会自动创建所需目录，但也可以手动创建以确保权限正确：

```bash
# 在项目根目录（虚拟环境激活状态下）
mkdir -p data/五池管理 data/历史记录 data/cache data/ml_model logs
mkdir -p data/复盘记录 data/回顾报告 data/运营日报
```

> 💡 这些目录已加入 `.gitignore`，不会提交到 Git。

### 3.7 验证部署

```bash
# 1. 编译验证（确保所有模块语法正确）
python -m py_compile main.py
python -m py_compile agents/screen_agent.py
python -m py_compile agents/review_agent.py
python -m py_compile agents/decision_agent.py
python -m py_compile agents/skeptic_agent.py
python -m py_compile agents/pool_manager.py

# 2. 健康检查

# 3. 查看五池状态

# 4. 执行全流程（可选，首次部署建议用 news 测试）
```

**预期输出：**

- `health` 命令：显示各模块运行状态，无报错
- `status` 命令：显示五池 JSON 文件存在性及条目数
- `news` 命令：生成新闻分析报告，无 API 调用错误

## 五、常见问题

### 5.1 依赖安装失败

**现象：** `pip install -r requirements.txt` 报错

**排查步骤：**

1. 确认 Python 版本 ≥ 3.11
2. 确认虚拟环境已激活
3. 升级 pip：`pip install --upgrade pip`
4. 逐行安装排查：`pip install requests && pip install pandas && ...`
5. 检查网络代理：`no_proxy=pypi.org pip install -r requirements.txt`

### 5.2 LLM API 调用失败

**现象：** `python main.py news` 报 API 相关错误

**排查步骤：**

1. 确认环境变量已配置：`echo $OPENCODE_ZEN_API_KEY`
2. 确认 API 端点可访问：`curl -s --connect-timeout 10 https://opencode.ai/zen/v1/chat/completions`
3. 检查 API Key 是否有效（可能过期或额度用完）
4. 检查代理设置：`http_proxy=` 和 `https_proxy=` 环境变量是否干扰
5. 尝试切换后端：在 `config.yaml` 中修改 `api.llm.backend` 为 `sensenova` 或 `opencode`

### 5.3 新闻源拉取失败

**现象：** 新闻分析报告为空或超时

**排查步骤：**

1. 检查 DuckBurn GitHub 可访问性（首选数据源）：
   ```bash
   curl -sL --connect-timeout 10 --max-time 30 "https://raw.githubusercontent.com/DuckBurnIncense/xin-wen-lian-bo/master/news/$(date +%Y%m%d).md"
   ```
2. 检查 mrxwlb.com 是否超时（该源已标记为不稳定）
3. 检查系统代理：`http_proxy= https_proxy= curl ...` 绕过代理重试
4. 降级为东方财富 fallback 数据源

### 5.4 代理导致 GitHub 连接失败

**现象：** `git clone` 或 `curl` 访问 GitHub 报 SSL 超时

**解决方案：**

```bash
# 临时绕过代理
no_proxy=github.com NO_PROXY=github.com http_proxy= https_proxy= git clone https://github.com/weis1655/tianshu-quanheng.git

# 或设置 Git 专用代理
git config --global http.https://github.com.proxy ""
```

### 5.5 Hermes Cron 任务未执行

**现象：** 定时任务没有自动运行

**排查步骤：**

1. 检查 Cron 注册状态：`hermes cron list`
2. 检查 agent.log 中 scheduler 日志（`Running job` / `completed successfully`）
3. 确认 Hermes Agent 后台运行中
4. 注意调度器串行阻塞：一个长任务会阻塞后续所有任务
5. 手动补跑：`python main.py full`

### 5.6 模块编译报错

**现象：** `python -m py_compile` 报语法错误

**常见原因：**

1. Python 版本过低（需要 3.11+）
2. 缺少依赖包（`from X import Y` 报 ModuleNotFoundError）
3. 文件编码问题（非 UTF-8 编码的源文件）
4. 不一致的缩进（混合 tab 和空格）

### 5.7 股票行情数据获取失败

**现象：** `status` 命令或审查阶段行情数据为空

**排查步骤：**

1. 确认系统时间正确（港股/美股行情使用北京时间）
2. 确认非交易日（周六日/节假日行情数据不更新）
3. 检查网络连通性：`curl -s --connect-timeout 5 "https://hq.sinajs.cn/list=sh000001"`
4. 确认行情数据源配额未超限
