> **维护人:** 七郎

## 概述

本文档描述天枢权衡系统的五池流转体系——覆盖快筛候选池、重点观察池、S级操作池、边缘池、持仓池五个股票池的容量限制、流转规则、升级/降级/淘汰阈值、评分衰减、三振出局、跨池去重、S级记事本模式等核心机制。所有阈值统一以 `agents/thresholds.py` 为单一真相来源（SSOT），代码中不应存在硬编码阈值。

## 五池总览

| 池名称 | 容量 | 定义 | 数据文件 |
|--------|:----:|------|----------|
| 快筛候选池 | 20 | 收纳快筛初选、尚未审查的股票 | `五池管理/快筛候选池.json` |
| 重点观察池 | 20 | 审查通过、值得重点跟踪的对象 | `五池管理/重点观察池.json` |
| S级操作池 | 3 | 当日决策主推标的，T+0可追，T+1需评估 | `五池管理/S级操作池.json` |
| 边缘池 | 30 | 审查后降级、暂不符合条件的对象 | `五池管理/边缘池.json` |
| 持仓池 | 不限 | 当前真实持仓 | `五池管理/持仓池.json` |

> **容量配置来源（P2-1）**：优先从 `config.yaml` 的 `pools.capacity` 加载，失败时回退至 `thresholds.py` 的 `POOL_CAPACITY_LIMITS` 字典。

### 辅助池

| 池名称 | 容量 | 说明 |
|--------|:----:|------|
| 重点观察池_历史池 | 200 | 重点观察池溢出的历史记录归档 |

## 流转总图

```
新闻联播 → 快筛 → 快筛候选池 → 审查 → 重点观察池 → 决策 → 持仓池
                                   ↙
                              S级操作池 ──(优先读取)→ 决策层
                                                      ↓
                                                复盘层（T+1/T+3/T+5）

快筛候选池 ──(评分<65)──→ 边缘池
重点观察池 ──(评分<65)──→ 边缘池
S级操作池   ──(T+1过期)──→ 自动清理
边缘池      ──(>30天/评分<40)──→ 彻底归档
边缘池      ──(评分>60回流)──→ 快筛候选池
```

## 评分等级体系

评分等级定义了所有池间流转的基准，所有阈值统一在 `thresholds.py` 中管理。

| 分数 | 等级 | 标签 | 说明 |
|------|:----:|------|------|
| ≥90 | S级 | — | 极佳机会 |
| 75-89 | A级 | 升级 | 审查升级门槛，可制定执行方案 |
| 65-74 | B级 | 黄色预警 | 谨慎观察，暂不列入推荐 |
| 55-64 | C级 | 观察区 | 暂缓，建议降级至边缘池 |
| <55 | D级 | 淘汰 | 一票否决，强制降级 |

### 关键阈值常量

| 常量名 | 值 | 用途 |
|--------|:---:|------|
| `SCORE_S_LEVEL` | 90 | S级起点 |
| `SCORE_A_LEVEL` | 75 | 审查升级门槛 / 可关注 |
| `SCORE_B_LEVEL` | 65 | 黄色预警下限 |
| `SCORE_C_LEVEL` | 55 | 观察区下限 |
| `HARD_DOWNGRADE_SCORE` | 60 | <60分→强制降级边缘池 |
| `AUTO_DOWNGRADE_SCORE` | 65 | <65分→存量扫描自动降级 |
| `DECISION_MIN_SCORE` | 75 | 决策执行最低分 |
| `S_POOL_MIN_SCORE` | 75 | S级操作池准入分 |
| `KEY_WATCH_MIN_SCORE` | 50 | 重点观察池准入分 |
| `ONE_VETO_MAX_SCORE` | 54 | 一票否决降级上限 |

> 💡 **v5.94 校准**：A级升级门槛从 70→75。实测原70-74分段实战准确率仅25%（评分与表现倒挂），旧阈值的"建议关注"标的大部分实际亏损。

## 各池详细规则

### 1. 快筛候选池

**容量**: 20只

**进入条件**:
- 新闻联播快筛阶段（`screen_agent.py`）输出S/A/B/C级驱动对象
- 技术面补位扫描捕获的量价异动股（涨幅>5%+换手>8%+量比>1.5+振幅>5%+市值>50亿，最多推送5只，标注B级）
- 边缘池评分>60的回流标的

**流出路径**:
| 目标池 | 条件 | 触发者 |
|--------|------|--------|
| 重点观察池 | 审查评分≥75 | `review_agent.py` 审查升级 |
| 边缘池 | 审查评分<65，或评分0且入池≥3天 | `review_agent.py` 降级 + `pool_manager._scan_and_downgrade()` |
| 淘汰 | 入池>14天未被审查 | `pool_manager.clean_expired_candidates()` |

**超限处理**: 保留最近20只，自动截断，记录告警日志。

### 2. 重点观察池

**容量**: 20只

**进入条件**:
- 审查评分≥75的标的从快筛候选池升级
- 评分≥50（`gate_controller.enforce_writing_rules` 准入规则）

**流出路径**:
| 目标池 | 条件 | 触发者 |
|--------|------|--------|
| 持仓池 | 决策Agent生成建仓方案，执行建仓 | `pool_manager.add_to_holding()` |
| S级操作池 | 决策Agent写入【主推】标的 | `pool_updater.update_s_pool()` |
| 边缘池 | 评分<65（存量扫描降级），或评分<60（硬性降级），或评分0且入池≥3天，或SkepticGate三振出局 | `pool_manager._scan_and_downgrade()` + `gate_controller.check_blocked_count()` |
| 重点观察池_历史池 | 超容量溢出后自动归档 | `pool_manager.save_pool()` |

**池内维护**:
- 实时价格刷新（`refresh_key_watch_prices()`）：每15分钟（09:00-14:50）拉取行情，更新今日收盘/涨跌幅/换手率/量比/振幅
- 评分时间衰减（详见下文）
- 止损检查：收盘价<止损线→标记"已跌破止损，建议调出"
- 信心度补填（`enrich_confidence_for_existing_stocks()`）：基于综合分自动推算（≥80→高，70-79→中高，60-69→中，<60→低）
- 按综合分降序自动排序，无分值的排最后

### 3. S级操作池

**容量**: 3只（P2升级：原2只）

**进入条件**:
- 决策Agent（`decision_agent.py`）在决策报告中标记【主推】标的
- 走"记事本模式"直接写入，不进行二次审查

**记事本模式（2026-06-30 重构）**：

S级操作池原设计为"门卫模式"——配合三道防线（准入评分/S_POOL_MIN_SCORE=80、QualityGate质检门、52周高位拦截）二次审查决策Agent的推荐。实际运行18天空池（6/12后无任何标的进入），根因为决策Agent和SkepticGate已跑完整流程，S池二次审查重复拦截。

重构为"记事本模式"后，决策Agent的决定直接记录入池，不做二次审查：

```
旧（门卫模式）：决策报告 → 防线一(评分检查) → 防线二(价格检查) → 防线三(QualityGate) → S池
新（记事本模式）：决策报告 → PoolUpdater(仅价格检查) → S池
```

**保留的检查**：
- ✅ **52周高位拦截**：当前价距52周高点>92%且MA5≤MA10（非上升趋势）时拒绝入池，防追高。P1-3升级：上升趋势(MA5>MA10)中的高位=强势股放行，不拦截；API无MA数据时视为无法判断趋势，放行
- ✅ **市场状态容量限制**：偏多3只/震荡偏强3只/震荡2只/震荡偏弱2只/偏空1只（P2升级：偏空保留1只，原0只）
- ✅ **跨池去重**：入S池时自动从重点观察池移除原标的
- ✅ **评分字段提取**：从决策报告正则提取综合评分（`_extract_score()` 方法，0次LLM调用）
- ✅ **入场价提取优化**（2026-07-21）：S池入场价不再使用实时行情，改为从决策报告正则提取推荐买入价。格式：`买入价：5.95` 或 `买入区间 5.95-6.00`

**移除的检查**（已下沉到决策Agent+SkepticGate）：
- ❌ 防线一：scored_stocks评分检查
- ❌ 防线三：QualityGate质检门（历史表现+过热检测）

**流出路径**:
| 目标池 | 条件 | 触发者 |
|--------|------|--------|
| T+1过期清理 | 入池>1天（T+1过期） | `pool_manager.clean_expired_s_pool()` |
| 边缘池 | 跌破止损线 | `pool_manager.refresh_key_watch_prices()` 中的S级止损检查 |
| 边缘池 | 全池低分扫描评分<65 | `scripts/sweep_downgrade.py`（独立于main.py的降级扫描） |

**T+1过期机制**：
- `pool_manager.clean_expired_s_pool(max_age_days=1)` 每天运行
- 入池超过1天的标的自动移出（保留到历史记录中）
- 历史记录保留在 `S级操作池.json` 的 `历史记录` 数组中，供后续命中率评价

**S级操作池今日保护（2026-07-21）**：
- 今日刚入S级操作池的标的获得1天保护期，不被 `sweep_downgrade` 清理
- 保护期判断：`纳入日期 == 今日日期`

**S级操作池历史评价**：
- `pool_manager.evaluate_s_pool_history()` 读取历史记录，查当前价计算涨跌幅
- 涨跌幅≥3%→命中，≥0%→微涨，≥-3%→微跌，<-3%→偏差
- 输出命中率、平均涨跌幅、逐条明细

### 4. 边缘池

**容量**: 30只（P2升级：原20只）

**进入条件**（三条降级路径）:
| 来源 | 触发条件 | 写入字段 |
|------|---------|----------|
| 审查降级 | 审查评分<65，或硬性降级<60，或评分0且入池≥3天 | 代码/名称/综合分/降级时间/降级原因 |
| 存量扫描降级 | `pool_manager._scan_and_downgrade()` 扫描发现评分<65 | 同上 |
| SkepticGate三振 | 连续阻塞≥2次（原3次，2026-06-30调降为2次） | 同上 |
| 止损降级 | 重点观察池/S级操作池标的跌破止损线 | 同上 |

**流出路径**:
| 目标池 | 条件 | 触发者 |
|--------|------|--------|
| 快筛候选池 | 清理时评分>60的高评分标的回流 | `pool_manager.clean_expired_edge_pool()` |
| 彻底淘汰 | 入池>30天，或评分<40 | `pool_manager.clean_expired_edge_pool()` |

**超限处理**:
1. 先清理入池超过30天的过期标的，腾出空间
2. 仍超容量则保留最近30只，自动截断

> ⚠️ **边缘池数据完整性**：三条降级路径中必须全部携带 `综合分` 字段。2026-07-19 审计发现两条路径缺 `综合分`，已修复。所有写入边缘池必须携带 `综合分`。

### 5. 持仓池

**容量**: 不限

**进入条件**:
- 决策Agent生成建仓方案，调仓专员/系统执行建仓后，通过 `pool_manager.add_to_holding()` 写入

**持仓记录字段**:
| 字段 | 说明 |
|------|------|
| 代码/名称 | 股票标识 |
| 市场 | SH/SZ（自动从代码推断） |
| 成本价/建仓日期 | 建仓成本和时间 |
| 盈亏额/盈亏比例 | 实时浮动盈亏 |
| 今日收盘/今日涨跌 | 实时行情 |
| 止损线/第一止盈/第二止盈 | LLM评估的止损止盈目标 |
| 操作建议 | 持有/减仓/清仓 |

**流出路径**:
| 目标 | 条件 | 触发者 |
|------|------|--------|
| 历史持仓 | 卖出（止盈/止损/失效） | 手动操作或系统止损 |
| 彻底归档 | 卖出后不再回到任何池 | 决策日志标记最终结果 |

**建仓流程**:
1. 检查是否已在持仓池中（去重）
2. 拉实时行情
3. LLM评估止损线/第一止盈/第二止盈/操作建议
4. 写入持仓池（含入池记录）

## 评分时间衰减

**规则**（`pool_manager.refresh_key_watch_prices()` 中无条件执行）:

| 参数 | 值 | 说明 |
|------|:---:|------|
| 衰减起始 | 入池>7天 | `SCORE_DECAY_DAYS = 7` |
| 衰减速率 | 0.5分/天 | `SCORE_DECAY_PER_DAY = 0.5` |
| 衰减上限 | 15分 | `SCORE_DECAY_MAX = 15` |
| 衰减下限 | 40分 | `SCORE_DECAY_FLOOR = 40` |

**公式**:
```
衰减量 = min(入池天数 × 0.5, 15)
今日上涨时 → 衰减量 × 0.5（今日涨跌幅含"+"标记时减半衰减）
新评分 = max(round(原始评分 - 衰减量), 40)
```

**风险提示**:
- 衰减基于衰减前的原始分数还是衰减后的分数？2026-07-09 发现旧实现存在"衰减级联 bug"——每轮衰减基于衰减后的分数再次衰减（而非基于原始分数），导致标的短期内评分骤降。修复后改为基于原始分数单次衰减。
- 判据：`ref: score-decay-cascade-bug-20260709.md`

## 存量低分降级扫描

### pool_manager._scan_and_downgrade()

扫描池中所有标的，对评分<65的自动降级至边缘池：

| 条件 | 处理 |
|------|------|
| 综合分 ≤ 0 且入池≥3天 | 强制降级边缘池（WO-002：不等待自然衰减） |
| 综合分 < 65（AUTO_DOWNGRADE_SCORE） | 强制降级边缘池 |
| 操作建议="已跌破止损，建议调出" | 止损降级边缘池 |

### sweep_downgrade.py（独立降级扫描）

2026-07-07 新增，独立于main.py的全池低分扫描，覆盖：
- 重点观察池
- 快筛候选池
- S级操作池（今日入池的标的除外——1天保护期）

评分<65强制降级至边缘池。支持 `--dry-run` 预览模式。集成到 main.py F2段 + cron job 15:05。

## 三振出局规则

### SkepticGate 阻塞计数

标的在SkepticGate（质疑审查Gate）中被连续阻塞的计数规则：

| 规则 | 值 | 说明 |
|------|:---:|------|
| 三振出局阈值 | 3次 | `SKEPTIC_BLOCK_LIMIT = 3`，连续阻塞≥3次自动降级边缘池 |
| 历史变更 | 3→2 | 2026-06-30 财经专家审计调降为2次，根因：无新催化事件的票被Gate持续否决，重复审查浪费LLM token。后恢复为3次 |
| 阻塞计数幂等性 | 同一天只计1次 | 2026-06-18 修复：`last_blocked_date` 字段防重复递增 |

### 阻塞时间衰减（P1-2，2026-07-01）

解决"旧质疑永不失效"问题：

- 14天无新阻塞 → 计数-1（最低保留1次）
- 累计阻塞≥5次且跨越60天以上 → 强制三振（护城河规则）
- 新增 `first_blocked_date`、`total_blocked_count` 字段追踪累计历史

### 首次high豁免（P1-2）

- 首次被SkepticGate阻塞（blocked_count=0）且非veto级别 → 只标记为"观察中"，不阻断
- 从第二次high开始才累加阻塞计数

### 阻塞计数重置

- 标的通过SkepticGate（不在阻塞列表中）→ 计数重置为0
- `last_blocked_date` 同步清除

## 跨池去重约束

### 核心规则

S级操作池与重点观察池必须互斥，同一只股票不能同时存在于两池。

### 三层防线

| 层 | 位置 | 机制 | 状态 |
|:-:|------|------|:----:|
| P0 | `review_agent.py` 的 `all_trade_pools` | 升级重点池时查不到S级池 → 2026-06-04 修复：S级池名加入 `all_trade_pools` | ✅ 已修复 |
| P1 | `pool_updater._check_s_pool_overlap()` | 晋级S级时自动从重点观察池/快筛候选池移除原标的 | ✅ 已修复 |
| P2 | `gate_controller.enforce_writing_rules()` | 跨池重复默认拦截（除非 `allow_cross_pool=True`） | ✅ 已修复 |

### 覆盖检查

`gate_controller.check_cross_pool_duplicate()` 遍历所有5个池（快筛候选池/重点观察池/S级操作池/边缘池/持仓池），返回存在该股票的所有池名列表。

## 评分0强制降级

**WO-002 修复（2026-07-07）**：

当V2正则提取评分失败（score=0）时，LLM原始输出中的 `flow_direction`（可能含"升级"）被沿用，导致score=0的标的被"升级"入重点观察池。

**修复**：在 `review_agent.py` 的 `_parse_review_result_v2` 中，若 `score == 0 and flow_dir != "降级"`，强制设 `flow_dir="降级"` + `target_pool="边缘池"`。阻断评分提取失败的标的流入升级列表。

**外层兜底**：`pool_manager._scan_and_downgrade()` 中，评分≤0且入池≥3天的标的，直接强制降级边缘池，不等待自然衰减。
