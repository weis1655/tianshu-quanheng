## 概述

本文档描述天枢权衡（Tianshu Quanheng）系统的整体架构，涵盖串行流水线设计、五池流转规则、Agent间契约、数据流方向以及各阶段之间的依赖关系。天枢是一个多Agent协同的A股选股与决策分析系统，以新闻驱动为起点，经多轮LLM分析与规则引擎审查，最终生成结构化投资决策方案。

## 二、核心Agent一览

| 组件 | 职责 | 输入 | 输出 | 关键文件 |
|------|------|------|------|---------|
| **NewsAgent** | 多源新闻抓取与分析，提取S/A/B/C级催化剂 | 新闻联播、东方财富、新浪新闻 | 新闻分析报告（含驱动分级） | `agents/news_agent.py` |
| **MarketAgent** | 实时行情数据获取（技术因子计算） | 五池股票代码列表 | 行情数据+技术面评分 | `agents/market_agent.py` |
| **ScreenAgent** | 新闻驱动快筛+技术面补位扫描 | 新闻分析报告、行情数据 | 快筛候选池、快筛报告 | `agents/screen_agent.py` |
| **ReviewAgent** | 四维深度审查（驱动/位置/量能/风险） | 快筛候选池、行情数据 | 审查报告+池间流转 | `agents/review_agent.py` |
| **SkepticAgent** | 五维质疑审查+加权阻塞裁决 | 重点观察池+S级操作池、审查报告 | 质疑审查报告+裁决JSON | `agents/skeptic_agent.py` |
| **DecisionAgent** | 决策方案生成（S池写入+决策报告） | S级操作池、重点观察池、质疑裁决 | 决策报告+执行方案 | `agents/decision_agent.py` |
| **PoolManager** | 五池CRUD+价格刷新+过期清理+降级扫描 | 各池JSON文件、行情数据 | 更新后的池JSON | `agents/pool_manager.py` |
| **GateController** | 跨池写入守卫+阻塞计数+去重约束 | 池数据、Skeptic裁决 | 过滤后的股票列表 | `agents/gate_controller.py` |
| **PoolUpdater** | S级操作池写入（记事本模式） | 决策结果、行情数据 | 更新后的S级操作池 | `agents/pool_updater.py` |
| **FeedbackLoop** | 收盘后反馈闭环（胜率统计+权重调整） | 决策日志、行情数据 | 反馈报告+权重调优历史 | `agents/feedback_loop.py` |
| **WeeklyReviewAgent** | 周复盘（池卫生+假设验证+权重修正） | 五池数据、决策日志 | 周复盘报告 | `agents/weekly_review_agent.py` |
| **ComplianceManager** | 合规检查（ST/涨跌停/T+1/举牌线等） | 交易指令、持仓数据 | 合规日志+拦截结果 | `agents/compliance_manager.py` |

## 三、Agent间契约

### 3.1 数据流依赖关系

```
NewsAgent  → 新闻分析报告(.md)          → ScreenAgent
ScreenAgent → 快筛候选池(.json)          → ReviewAgent
ReviewAgent → 审查报告(.md) + 池流转     → SkepticAgent
SkepticAgent → 质疑审查报告(.md) + 裁决JSON → DecisionAgent
DecisionAgent → 决策报告(.md) + S池更新  → 用户/持仓池
```

### 3.2 关键文件接口

| 文件 | 写入方 | 读取方 | 格式 |
|------|--------|--------|------|
| `五池管理/快筛候选池.json` | ScreenAgent/PoolManager | ReviewAgent | JSON (stocks数组) |
| `五池管理/重点观察池.json` | ReviewAgent/PoolManager | SkepticAgent/DecisionAgent | JSON (stocks数组) |
| `五池管理/S级操作池.json` | PoolUpdater | DecisionAgent | JSON (stocks数组) |
| `五池管理/边缘池.json` | PoolManager | 定期清理 | JSON (stocks数组) |
| `五池管理/持仓池.json` | PoolManager | FeedbackLoop | JSON (stocks数组) |
| `data/历史记录/{日期}_新闻联播投资分析.md` | NewsAgent | ScreenAgent | Markdown |
| `data/历史记录/{日期}_审查报告.md` | ReviewAgent | SkepticAgent | Markdown |
| `data/历史记录/{日期}_质疑审查报告.md` | SkepticAgent | DecisionAgent | Markdown |
| `data/历史记录/{日期}_质疑审查裁决.json` | SkepticAgent | DecisionAgent | JSON (裁决结构) |
| `data/历史记录/{日期}_决策报告.md` | DecisionAgent | 用户/邮件 | Markdown |
| `data/decision_log.json` | DecisionAgent | FeedbackLoop/回头看 | JSON (记录数组) |

### 3.3 异常处理契约

| 环节 | 异常行为 | 影响范围 |
|------|---------|---------|
| 新闻分析失败 | 质量门控拦截，终止全流程 | 全流程 |
| 快筛失败 | 级联终止，不进入审查 | 审查+后续全部 |
| 审查失败 | 级联终止，不进入Skeptic | Skeptic+决策+后续 |
| Skeptic失败 | 宪法守卫拦截，跳过决策 | 决策阶段 |
| 决策失败 | 池清理+准确率更新仍执行 | 无决策报告 |

### 3.4 熔断器保护

每个阶段（news_only/screen/review/skeptic/decision）均有独立熔断器，基于连续失败次数自动跳过来保护token消耗。熔断器状态持久化在 `data/circuit_breaker_state.json`。

## 四、五池流转概览

### 4.1 五池结构

```
                          ┌──────────────┐
                          │ S级操作池     │  ← 记事本模式，≤3只，T+1过期
                          │ (优先读取)    │
                          └──────┬───────┘
                                 │ 优先读取
                                 ▼
快筛候选池 ──(审查≥75)──▶ 重点观察池 ──(决策买入)──▶ 持仓池
     │                          │                       │
     │  (审查<65)               │ (Skeptic三振/         │ (止损/止盈/失效)
     │                          │  评分<65/时间衰减)    │
     ▼                          ▼                       ▼
    ┌──────────────────────────────────────────────────┐
    │                    边缘池                          │
    │  (评分<65降级标的，>30天无改善→彻底归档)          │
    └──────────────────────────────────────────────────┘
```

### 4.2 池容量限制

| 池名称 | 容量上限 | 超限处理 |
|--------|:-------:|---------|
| 快筛候选池 | 20 | 保留前20个，截断告警 |
| 重点观察池 | 20 | 保留最新20个，截断告警 |
| 重点观察池_历史池 | 200 | 溢出自动归档 |
| 边缘池 | 20 (config.yaml) | 保留最新20个，超容丢弃 |
| S级操作池 | 3 | T+1过期自动清理(原2只，P2升级为3只) |
| 持仓池 | 不限 | — |

### 4.3 评分阈值与流转规则

| 分数 | 级别 | 流向 | 说明 |
|:----:|:----:|------|------|
| ≥90 | S级 | 升级→重点观察池 | 极佳机会 |
| 75-89 | A级 | 升级→重点观察池 | 可以关注 (v5.94升级门槛从70→75) |
| 65-74 | B级(黄) | 保留候选池 | 谨慎观察 |
| 55-64 | C级 | 降级→边缘池 | 暂缓 |
| <55 | D级 | 淘汰 | 一票否决 |

### 4.4 关键流转触发条件

| 流转路径 | 触发条件 | 数据来源 |
|---------|---------|---------|
| 快筛候选池→重点观察池 | ReviewAgent审查评分≥75 | `review_agent.py` 评分解析 |
| 重点观察池→S级操作池 | DecisionAgent记事本模式写入+价格检查 | `pool_updater.py` |
| 任何池→边缘池 | 评分<65 (降级扫描) | `sweep_downgrade.py` / ReviewAgent |
| S级操作池→清理 | T+1过期 | `pool_manager.py` |
| 边缘池→彻底归档 | >30天无改善 | `pool_manager.py` |
| 持仓池→边缘池 | 止损触发 | `pool_manager.py` |

### 4.5 跨池约束

- **S级操作池与重点观察池互斥**：同一只股票不能同时存在于两池。晋级S级时自动从重点观察池移除。
- **GateController跨池去重**：默认拦截跨池重复写入（除非 `allow_cross_pool=True`）。
- **快筛候选池与边缘池反向交集**：晋级时自动从边缘池反向移除。

## 五、数据流方向

### 5.1 数据输入

```
外部数据源
  ├── 新闻联播 (mrxwlb.com / DuckBurnIncense GitHub / CCTV / 东方财富)
  ├── 实时行情 (新浪财经/腾讯API)
  ├── 技术因子 (6因子：MA5乖离/MA10乖离/RET5/RET20/VOL20/量比)
  └── 财报数据 (PE/换手率/市值等)
          │
          ▼
    天枢权衡流水线
```

### 5.2 数据流向（full_cycle模式）

```
阶段1: [交易日守卫] ──────────────── 非交易日/周末 → 仅新闻+池维护
阶段2: [新闻分析] ── NewsAgent读取外部新闻源，生成分析报告
阶段3: [质量门控] ── 新闻质量不合格 → 终止
阶段4: [市场行情] ── MarketAgent拉取实时行情+技术因子
阶段5: [池价格刷新] ── PoolManager刷新4个池的最新价格
阶段6: [T+1追踪] ── ClosedLoopTracker回溯昨日推荐标的T+1表现
阶段7: [快筛] ── ScreenAgent读取新闻+行情，筛选候选股→写入快筛候选池
         │ 失败 → 级联终止
阶段8: [审查] ── ReviewAgent读取快筛候选池，四维评分
         │ 失败 → 级联终止
         │ 评分≥75 → 升级至重点观察池
         │ 评分<65 → 降级至边缘池
阶段9: [截断检测] ── 检查审查报告的完整性
阶段10: [市场状态判断] ── 5档市场状态决定走SkepticLLM还是弱市简化
         ├── 偏多/震荡偏强/震荡 → 阶段11a: SkepticAgent(LLM)
         └── 震荡偏弱/偏空 → 阶段11b: 弱市简化审查(规则引擎)
阶段11: [质疑审查] ── 产出质疑审查报告+裁决JSON
         │ Skeptic失败 → 宪法守卫拦截决策
阶段12: [决策] ── DecisionAgent读取S池+重点池，生成决策报告
         │ 输出: 决策报告.md + S级操作池更新 + 决策日志
阶段13: [池清理] ── 候选池过期清理 + 边缘池过期清理
阶段14: [条件单扫描+合规日结] ── 实盘止损集成
阶段15: [准确率刷新+MemPalace保存] ── 持久化执行结果
```

### 5.3 数据输出

```
决策报告 (.md) ───▶ 用户/飞书卡片/邮件
质疑审查报告 (.md) ──▶ 决策Agent输入
五池状态 (.json) ──▶ 决策Agent/用户查询
决策日志 (.json) ──▶ 反馈闭环/回头看/准确率计算
合规日志 (.jsonl) ──▶ 合规审计
MemPalace (三元组) ──▶ 跨天记忆
```

## 六、关键规则体系

### 6.1 评分体系权重

| 维度 | 权重 | 关键规则 |
|------|:----:|----------|
| 驱动分析 | 25% | — |
| **位置分析** | **35%** | 处于52周高位(>80%分位)至少扣10分 |
| 量能分析 | 20% | — |
| 风险分析 | 20% | PE<0(亏损)→风险90分一票否决 |

### 6.2 过热检测

| 级别 | 条件 | 处理 |
|:----:|------|:----:|
| 🔥 CRITICAL | 涨幅>12% + (PE>80或换手>12%) | 强制降级边缘池，扣30分 |
| 🔥 CRITICAL | 月涨>25% + 评分≥70 | 强制降级边缘池，扣30分 |
| 🔥 CRITICAL | 季涨>50% | 强制降级边缘池，扣30分 |
| ⚠️ WARNING-1 | 涨幅>8% + 评分>75 | 扣10分 |
| ⚠️ WARNING-2 | 涨幅≥10% | 扣5分（强市豁免） |
| ⚠️ WARNING-3 | 涨幅>5% + 量比>3 | 扣5分（强市豁免） |
| ⚠️ WARNING-4 | 振幅>7% + 月涨>15% + 评分≥70 | 扣10分（强市豁免） |

### 6.3 评分时间衰减

入池 > 7天开始衰减，0.5分/天，最低保留至55分。

### 6.4 三振出局规则

标的连续**2次**被SkepticGate阻塞，自动从重点观察池移入边缘池。阻塞计数带时间衰减：
- 14天无新阻塞 → 计数-1（最低保留1次）
- 累计阻塞≥5次且跨越60天以上 → 强制三振
