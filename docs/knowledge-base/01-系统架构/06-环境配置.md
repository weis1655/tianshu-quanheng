## 一、Python 版本要求

| 项目 | 要求 |
|------|------|
| Python 版本 | **3.11+** |
| 包管理器 | pip（标准）或 uv（推荐） |

## 三、环境变量配置（.env 文件）

### 3.1 文件位置

系统不要求项目目录下的 `.env` 文件。实际 .env 加载路径为 **`~/.hermes/.env`**（Hermes 配置目录），由 `main.py` 第 45-62 行自动加载：

```python
hermes_home = os.path.expanduser("~/.hermes")
env_path = os.path.join(hermes_home, ".env")
if os.path.exists(env_path):
    # 优先使用 dotenv，fallback 到手动解析
```

### 3.2 核心环境变量

| 变量名 | 必填 | 说明 | 示例值 |
|--------|:----:|------|--------|
| `OPENCODE_API_KEY` | **是** | OpenCode API 密钥 | `sk-xxxxxxxxxxxxxxxx` |
| `http_proxy` | 按需 | HTTP 代理地址 | `http://192.168.197.109:7897` |
| `https_proxy` | 按需 | HTTPS 代理地址 | `http://192.168.197.109:7897` |
| `no_proxy` | 按需 | 不走代理的地址白名单 | `localhost,127.0.0.1,github.com` |
| `TIANSHU_PROJECT_ROOT` | 否 | 项目根目录路径（供外部脚本使用） | `/home/seven/hermes-data/tianshu-quanheng` |

### 3.3 配置加载顺序

系统使用 `config_loader.py`（`agents/config_loader.py`）中的 `ConfigLoader` 类加载配置，支持环境变量引用语法 `${VAR_NAME:-default_value}`：

1. 读取 `config.yaml` 原始值
2. 正则匹配 `${VAR_NAME}` 或 `${VAR_NAME:-default}` 模式
3. 从 `os.environ` 中取环境变量值替换
4. 未设置且无默认值 → 替换为空字符串

**示例：** `config.yaml` 中 `opencode_key: "${OPENCODE_API_KEY:-***}"` 在运行时会被替换为 `OPENCODE_API_KEY` 环境变量的实际值，若未设置则显示 `***`。

## 四、API 凭据配置

### 4.1 OpenCode API

| 参数 | 配置位置 | 说明 |
|------|---------|------|
| URL | `config.yaml` → `api.opencode_url` | API 端点地址 |
| 密钥 | 环境变量 `OPENCODE_API_KEY` | 从 `config.yaml` 引用 `${OPENCODE_API_KEY}` |
| 默认模型 | `config.yaml` → `api.default_model` | 当前值：`minimax-m2.5-free` |

当前配置的 OpenCode 端点：

```yaml
api:
  opencode_url: "https://opencode.ai/zen/v1/chat/completions"
  opencode_key: "${OPENCODE_API_KEY:-***}"
  default_model: "minimax-m2.5-free"
```

### 4.2 LLM 后端选择

`config.yaml` 中的 `api.llm.backend` 字段控制后端选择，支持以下选项：

| 后端 | 值 | 说明 | 状态 |
|------|----|------|:----:|
| OpenCode | `opencode` | 默认 OpenAI 兼容 API | ✅ 可用 |
| Gemini | `gemini` | Google Gemini CLI | ❌ 不可用（CLI 环境问题，2026-05-15 确认） |
| 豆包 | `doubao` | 字节跳动豆包 | ❌ 不可用（触发 CAPTCHA，2026-05-15 确认） |
| 商汤 | `sensenova` | SenseNova 商汤 API | ✅ **当前首选**（最可靠，2026-05-15 切换） |

当前默认后端：`sensenova`

### 4.3 飞书 API

`config.yaml` 中 `features.feishu_notify: false` 表示飞书通知默认关闭。如需启用：

1. 在 `~/.hermes/.env` 中配置飞书应用凭据（`APP_ID`、`APP_SECRET`）
2. 将 `config.yaml` 中 `features.feishu_notify` 设为 `true`
3. 安装飞书 SDK（取消 `requirements.txt` 中 `feishu-sdk>=1.0.0` 的注释）

### 4.4 SenseNova API 配置说明

当后端设为 `sensenova` 时，系统通过 `base_agent.py` 中的 `BaseAgent` 类调用商汤 API，使用与 OpenCode 兼容的 OpenAI 协议格式。需确保：

- `OPENCODE_API_KEY` 环境变量已设置为有效的 SenseNova API Key
- API 端点可访问（无需额外配置）

## 五、config.yaml 参数说明

配置文件路径：`~/hermes-data/tianshu-quanheng/config.yaml`。由 `agents/config_loader.py` 中的 `ConfigLoader` 类加载，支持环境变量替换和点号分隔路径访问。

### 5.1 API 配置（`api` 模块）

```yaml
api:
  opencode_url: "https://opencode.ai/zen/v1/chat/completions"
  opencode_key: "${OPENCODE_API_KEY:-***}"
  default_model: "minimax-m2.5-free"
  llm:
    temperature: 0.3       # LLM 生成温度（低=确定性高）
    max_tokens: 1000       # 单次最大生成 token 数
    timeout: 60            # API 请求超时（秒）
    max_retries: 3         # 失败重试次数
    backend: "sensenova"   # 后端选择：opencode/gemini/doubao/sensenova
```

### 5.2 股票池配置（`pools` 模块）

```yaml
pools:
  capacity:
    快筛候选池: 20
    重点观察池: 20
    边缘池: 30
    S级操作池: 3
    # 持仓池无上限
  field_mapping:
    code: "股票代码"
    name: "股票名称"
    cost: "成本"
    price: "最新价"
    pnl: "盈亏"
    pnl_pct: "盈亏比例"
    date: "建仓日期"
    remark: "备注"
```

> **说明：** 池容量可从 `thresholds.py` 的 `POOL_CAPACITY_LIMITS` 同步，`config.yaml` 中的配置可热加载覆盖。

### 5.3 筛选规则配置（`screening` 模块）

```yaml
screening:

    pe_min: 0           # 最低市盈率
    pe_max: 100         # 最高市盈率
```

## 六、代理设置

### 6.1 标准代理配置

系统支持通过标准环境变量配置 HTTP 代理。在 `~/.hermes/.env` 中设置：

```bash
# HTTP 代理
http_proxy=http://192.168.197.109:7897
https_proxy=http://192.168.197.109:7897
no_proxy=localhost,127.0.0.1,github.com,raw.githubusercontent.com
```

### 6.2 代理注意事项

1. **代理与 GitHub 直连冲突**（2026-07-13 确认）：系统 HTTP 代理（`192.168.197.109:7897`）可能导致 GitHub 直连 SSL 超时，使 `web_extract`/`web_search`/`browser` 工具失效。此时需绕过代理访问 DuckBurn GitHub 数据源：

   ```bash
   http_proxy= https_proxy= curl -sL --connect-timeout 10 --max-time 30 \
     https://raw.githubusercontent.com/DuckBurnIncense/xin-wen-lian-bo/master/news/YYYYMMDD.md
   ```

2. **`no_proxy` 建议值**：建议将 `github.com`、`raw.githubusercontent.com` 加入 `no_proxy` 白名单，避免 SSL 超时问题。

3. **代理故障处理**：详见 `references/news-data-sources-20260519.md` 的「代理故障处理」章节。

### 7.2 健康检查

系统提供两种健康检查方式：

**方式一：通过 main.py 入口**

```bash
cd ~/hermes-data/tianshu-quanheng
python main.py health
```

该命令触发 `Orchestrator` 中的健康检查流程，验证配置文件、池目录、数据目录、日志目录、持仓池和 API 配置。

**方式二：直接运行健康检查模块**

```bash
cd ~/hermes-data/tianshu-quanheng
python agents/health.py
```

该方式会执行完整的 `HealthChecker.check_all()`，输出以下检查项：

| 检查项 | 验证内容 | 状态阈值 |
|--------|---------|---------|
| 配置文件 | `config.yaml` 存在性 + YAML 解析 + 必需字段（api/pools/logging） | 缺失→error，缺字段→warning |
| 池目录 | `data/五池管理/` 存在性 + 4个必需池文件 | 目录缺失→error，池文件缺失→warning |
| 数据目录 | `data/` + `data/历史记录/` + `data/cache/` | 目录缺失→error |
| 日志目录 | `logs/` 存在性 + 文件大小（>100MB 告警） | 自动创建→warning，>100MB→warning |
| 持仓池 | `data/五池管理/持仓池.json` 解析 | 文件缺失→warning，解析失败→error |
| API 可达性 | API URL 配置检查（不实际调用） | URL 未配置→warning |

健康检查结果同时保存到 `data/reports/health_check_YYYYMMDD_HHMMSS.json`，自动清理超过 14 天的历史报告。
