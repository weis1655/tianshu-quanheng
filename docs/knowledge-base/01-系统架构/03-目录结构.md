## 各目录详解

### 根目录关键文件

| 文件 | 行数 | 职责 |
|------|:----:|------|
| `main.py` | 1341 | 系统入口。支持子命令：`full`（全流程）、`news`（新闻）、`screen`（快筛）、`review`（审查）、`decision`（决策）、`ts`（时间序列）、`status`（池状态）、`weekly`（周复盘）。通过 `Orchestrator` 编排各 Agent 阶段。 |
| `tianshu_memory.py` | 362 | MemPalace 记忆系统集成。实现跨天连续记忆，将五池状态、执行摘要、历史决策写入知识宫殿。提供 `wake_up()` 和 `save_run_summary()` 接口。 |
| `config.yaml` | 200 | 集中配置文件。覆盖 API 配置（端点/密钥/模型）、池容量、筛选规则、调度时间窗口、市场熔断、权重、日志、路径、功能开关、多策略组合、自动迭代等。详见 ARCH-07。 |
| `mempalace.yaml` | 55 | MemPalace 记忆宫殿配置，定义记忆存储结构和规则。 |
| `SOUL.md` | 219 | 系统运行的根本原则文档，涵盖 Agent 角色定义、行为约束、伦理边界。 |
| `CLAUDE.md` | 46 | Claude Code 的指令配置文件，定义 Agent 协作方式和代码生成规范。 |
| `requirements.txt` | 35 | Python 依赖列表，包含核心库（requests, yaml, pandas, numpy, pytest 等）。 |
| `Dockerfile` | 57 | 基于 Python 3.11-slim 的容器构建文件。 |
| `docker-compose.yml` | 82 | 编排服务，定义应用容器、卷挂载、环境变量、端口映射。 |

### agents/ — 核心 Agent 模块

**规模：54 个 `.py` 文件，合计 24947 行。**

此为系统核心代码。按功能分组如下：

#### 管道 Agent（Pipeline Core）

| 文件 | 行数 | 职责 |
|------|:----:|------|
| `orchestrator.py` | 140 | 全流程编排器。按阶段串联 News → Screen → Review → Skeptic → Decision，管理执行顺序和异常处理。 |
| `news_agent.py` | 639 | 新闻分析 Agent。从新闻联播文字稿提取投资驱动信号，输出行业板块驱动分析。 |
| `screen_agent.py` | 738 | 快筛 Agent。基于新闻驱动因子进行快速候选筛选，合并技术面补位规则。 |
| `review_agent.py` | 1616 | 审查 Agent。四维深度审查（催化剂 25% + 位置 35% + 量能 20% + 风险 20%），含过热检测。 |
| `review_evo.py` | 461 | 审查演化模块，迭代审查逻辑改进。 |
| `review_scorer.py` | 195 | 审查评分器，实现评分计算逻辑。 |
| `skeptic_agent.py` | 688 | 质疑审查 Agent。五维质疑（业绩、估值、竞争、持仓、量价），加权阻塞机制。 |

## 关键配置路径

### 配置加载优先级

系统配置加载顺序（从高到低）：

1. **环境变量** — 覆盖 `config.yaml` 中的值（如 `OPENCODE_API_KEY`）
2. **`config.yaml`** — 根目录集中配置文件（200 行）
3. **`path_config.py`** — 路径配置中心动态计算路径
4. **`thresholds.py`** — 阈值常量，作为 `config.yaml` 池容量配置的回退值

### 核心路径一览

| 路径 | 用途 | 定义位置 |
|------|------|---------|
| `config.yaml` | 系统配置 | 根目录 |
| `mempalace.yaml` | 记忆宫殿配置 | 根目录 |
| `agents/path_config.py` | 路径配置中心 | `path_config.py` |
| `agents/thresholds.py` | 阈值常量 SSOT | `thresholds.py` |
| `五池管理/*.json` | 五池持久化 | `path_config.pool_dir` |
| `data/` | 运行时数据 | `path_config.data_dir` |
| `logs/` | 运行日志 | `path_config.logs_dir` |
| `data/历史记录/` | 报告输出 | `path_config.report_dir` |
| `data/portfolio/` | 组合管理数据 | `config.yaml` → `portfolio.data_dir` |

### 配置文件中可热加载的路径

`config.yaml` 中 `paths` 段落定义了以下可配置路径（相对于项目根目录）：

```yaml
paths:
  data_dir: "data"
  pool_dir: "data/五池管理"      # 实际运行时使用五池管理/
  history_dir: "data/历史记录"
  cache_dir: "data/cache"
  logs_dir: "logs"
  report_dir: "data/历史记录"
```

> **注意：** 池路径实际已迁移到项目根目录 `五池管理/`，`config.yaml` 中的 `pool_dir` 配置可能已过时，建议以 `path_config.py` 的 `pool_dir` 属性为准。
