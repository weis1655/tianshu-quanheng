# 天枢权衡 — AI 修复规范

## 修复原则
1. **最小改动**：只改 1-2 行，不重构
2. **禁止修改**：配置文件、依赖文件、测试文件
3. **验证要求**：编译通过 + 模块导入通过
4. **单问题上限**：改动不超过 50 行

## 禁止修改的文件
- `config/` 目录
- `.env` 及 `.env.*`
- `requirements.txt` / `pyproject.toml`
- `agents/__init__.py`

## 阈值参数保护区（禁止随意修改）
以下阈值经过多轮校准（v5.92/v5.94/v6.0），修改前必须人工确认：
- **审查升级阈值**：综合分≥75 → 升级重点观察池（`review_agent.py`）
- **决策准入阈值**：综合分≥75 → 可制定执行方案（`decision_agent.py` prompt）
- **S级准入阈值**：综合分≥75 → 可入S级操作池（`gate_controller.py`，与 `thresholds.S_POOL_MIN_SCORE` 对齐）
- **WARNING-1过热阈值**：日涨>8% + 综合分>75 → 扣10分（`review_agent.py` L1050，v5.94校准）
- **CRITICAL过热阈值**：月涨>25% + 综合分>=70 → 强制降级（`review_scorer.py` L91，2026-06-05修复>=边界）
- **黄色预警区间**：60-74分（`decision_agent.py` 多处，v6.0校准）
- **入池超14天淘汰**：候选池标的超过14天未升级则自动移除（`screen_agent.py`）

## 已知架构约束（需长期迭代，非立即修复）

> 以下项为系统架构层面的已知限制，由 2026-07-09 独立深度评测（P-H04/P-H05）识别。
> 标记在此作为长期迭代清单，不阻断当前运行。

### P-H04: 决策-执行断链（决策建议层 vs 实盘执行层）
- 现状：`DecisionAgent` 生成执行方案（买入价/止损/止盈），但**无实际券商接口执行下单**
- 影响：系统停留在"建议"层，T+1追踪仅记录涨跌幅，不构成完整交易闭环
- 建议：长期接入券商 API 或独立执行模块；过渡期手动执行后回填实际成交数据

### P-H05: 回测无滑点与冲击成本模拟
- 现状：`backtest_sandbox.py` 与回测流程**未模拟滑点与冲击成本**
- 影响：回测收益被系统性高估，实盘表现可能大幅低于预期
- 建议：在回测层引入滑点模型（如成交量×0.02%冲击成本）

### P-H06: 测试覆盖不足
- 现状：核心模块（decision/review/skeptic）零测试，仅 58/80+ 模块有覆盖
- 影响：重构/修复极易引入回归错误
- 建议：优先为核心 Agent 增加基础单元测试

## 修复流程
1. 定位问题 → 2. 最小改动 → 3. 编译验证 → 4. 提交 → 5. 等待 PR 审核
