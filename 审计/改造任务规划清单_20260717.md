## 定时任务统一规范

### 调度规则

| 维度 | 规范 |
|:----|:-----|
| 框架 | Hermes Cron Scheduler（非 `scripts/auto_tasks.py` 单独调度） |
| 周期格式 | 标准 cron 表达式（`分 时 日 月 周`） |
| 大盘开盘日 | `* 1-5` 仅工作日 |
| 大盘非开盘日 | `* 0,6` 周末可减量运行 |

### 任务状态管理

| 字段 | 说明 |
|:----|:-----|
| `last_run_at` | 上次执行时间 |
| `last_status` | `ok`/`error`/`timeout` |
| `last_delivery_error` | 最近投递错误（如有） |
| `enabled` | 是否启用 |

### 日志规范

- 脚本输出 `[任务名] OK/FAIL | 详细信息` 格式
- 运行日志自动存至 `data/auto_tasks/{task_name}.log`

### 异常重试机制

| 场景 | 策略 |
|:----|:-----|
| 网络超时 | 自动重试2次，间隔30s |
| API失败 | 报错日志记录，不重试（防重复操作） |
| 数据不一致 | 幂等检查，出现则报错终止 |
| 执行超时（180s） | 超时后标记 FAIL，下次Cron再跑 |

### 手动触发兜底

所有定时任务保留 CLI 入口：
```bash
python scripts/auto_tasks.py --task <name>   # 手动触发
python scripts/auto_tasks.py --all            # 全量执行
```
